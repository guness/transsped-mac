## TransSped v0.0.1 — first release

Log in to **ANAF SPV** from **macOS Firefox** using a **Trans Sped cloud**
qualified certificate — no Windows VM, no Wine, no physical token. This is the
macOS counterpart to the Windows EasySign / Cloud KSP.

**Signed with a Developer ID and notarized by Apple** — the DMG opens on any
Mac with no Gatekeeper warning.

### Install

1. Download **`TransSped.dmg`** below and open it.
2. **Quit Firefox**, then run **TransSped** (from the DMG, or drag it to
   Applications first).
3. Enter your **Trans Sped userID** (the email/phone for your cloud
   certificate). The app fetches your certificate and registers a PKCS#11
   module into your **normal** Firefox profile — no separate profile, no
   scripts, and it leaves your existing certificates alone.
4. Open Firefox as usual → your ANAF login → **certificate** method → pick
   **Trans Sped Cloud** → enter **PIN + OTP** in the single dialog (tick
   **Remember PIN** and later logins ask only for the OTP).

### Requirements

- macOS 11 or later
- Firefox (Chrome/Safari use the macOS Keychain and can't load a PKCS#11 module)
- A Trans Sped **cloud** qualified certificate already enrolled with ANAF
  (form 150)

### Uninstall

```bash
open "TransSped.app" --args -uninstall
```

Removes the module from Firefox and deletes `~/.config/tscloud` (and any
remembered PIN).

### Notes

- The working path is ANAF's F5 APM portal (`app.anaf.ro`), which requests
  `rsa_pkcs1_sha256`. The OAuth portal (`login.anaf.ro`) requests SHA-384,
  which the cloud can't sign, so use the certificate option that lands on the
  F5 portal.
- Signing is delegated to the Trans Sped cloud on every login; no private key
  is ever stored locally. A remembered PIN is kept in the macOS login Keychain.

and [RUNBOOK](https://github.com/guness/transsped-mac/blob/main/docs/RUNBOOK.md)
